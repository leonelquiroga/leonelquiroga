I don't write code first. I figure out what actually needs to be solved, map the possible paths, and pick the one that fits — not the most impressive one, the right one. A personal portfolio doesn't need a microservices cluster. A high-traffic ERP does.

I care about code the next person can read without asking questions. Consistent naming, decisions documented where they matter, no clever tricks that save three lines but cost an hour of confusion.

One thing I'm proud of: a **fully dynamic table renderer** — given a schema from the backend, it draws any table under any rules without touching the component. That kind of reusable precision is how I like to think about problems.
